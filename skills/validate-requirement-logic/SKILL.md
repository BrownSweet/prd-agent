# 需求逻辑校验

## 扫描方法

每轮重新扫描完整需求，不只复查上一轮问题：

1. 依赖关系：先后顺序、删除影响、循环依赖。
2. 状态流转：状态集合、转换条件、终态和回退。
3. 边界条件：上下限、空值、重复、并发和幂等。
4. 操作影响：已有数据、C端、其他模块以及生效时间。
5. 逆向操作：删除、取消、恢复、退款和补偿。
6. 交互一致性：弹窗、跳转、返回和二次确认。
7. 异常场景：网络、超时、空数据、下架、过期和服务错误。

## 严重程度

- `blocking`：不解决会导致主流程无法实现、数据错误或不可恢复损失。
- `important`：不解决会造成明显歧义、体验断点或高维护成本。
- `suggestion`：不影响当前正确性的可选优化。

保持 `L-XXX` 稳定编号。用户答案解决问题时，将决策同步到完整需求并记录
resolution。不要决定流程是否进入下一阶段。
