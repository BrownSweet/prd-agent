# PRD固定模板

按以下顺序使用二级标题：

1. 文档信息：版本、日期、负责人、状态。
2. 修订历史：版本、日期、修订人、内容。
3. 项目背景与目标：背景、目标、非目标、用户角色表。
4. 核心名词定义：名词、定义、示例。
5. 功能需求总览：模块、子模块、功能点、描述、P0/P1/P2。
6. 详细功能需求。
7. 非功能需求。
8. 全局交互规范。
9. 附录：状态定义、计算公式、待迭代清单。

每个详细功能必须包含：

- 字段规格表：字段名、类型、校验规则、默认值、说明。
- 交互步骤表：步骤序号、操作、系统反馈。
- 业务规则：唯一 `R-XXX` 编号。
- 页面状态：Loading、Empty、Error、Data。纯API功能说明不适用原因，
  并改为请求中、空响应、错误响应、成功响应语义。
